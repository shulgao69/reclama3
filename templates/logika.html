Корзина

Корзина работает на основе сессии. В рамках одной сессии может быть несколько клиентов. Например на одном ПК может зайти несколько пользователей с разными паролями. Их корзины будут различны.

Сессии

Просмотреть все сессии можно print(session)
Получим словарь
Пример: {'_fresh': True, '_id': '47b3eda5752ce2eafef....', '_user_id': '1'}
Создать запись в сессии: например session['card_usluga_id']=35
Получить значение из сессии: например session.get('_fresh', '')
Очистить всю сессию: session.clear()
Удалить запись в сессии: session.pop('card_usluga_id')

Карточки услуг

Карточка услуги содержит прайс, фото, и находится только в одном разделе и услуге.
Карточка услуги отражается на сайте только если имеет фото, активна, не в архиве.....
К одному разделу может относится много карточек услуг. Вероятнее всего у них будет одинаковый тип производства, но разное время (норматив) на выполнение заказа и разные ответственные лица (например печать книг и печать визиток). Эта специфика отражается в спецификациях карточек услуг, которые задаются при создании карточки услуги.

Статусы заказов

Статусы заказов - это общие этапы по заказу, который может состоять из нескольких разных услуг. Этапы не нормированы и отражают только общую динамику по заказу.
Согласно логике сайта статусы одинаковы для всех заказов. При изменении названий этапов (статусов) и их веса может поменятся последовательность и логика работы над заказом.
Поэтому при задании статусов заказов необходимо согласовать все и далее не предоставлять доступ к редактированию (за исключением топов).

Статусы карточек услуг

Статусы карточек услуг (StatusCard) - это стандартный набор статусов, общий для всех карточек услуг. Вес задавать так - 0, 1000, 2000 (Пока так). Нужно для того, чтобы потом сортировать в соответствии с основными статусами? Складывать вес основного статуса и промежуточного для сортировки? Затем для каждой карточки услуги задаются спецификации (SpecificationStatusCard) для каждого стандартного статуса карты. В этой спецификации задается временной норматив и ответственный (роль). Это позволяет контролировать сроки выполнения заказа и видеть на каком этапе происходит отклонение и кто за это отвечает.

Промежуточные статусы карточек услуг (зависят от статуса карт и типа производства)

Промежуточные статусы карточек услуг (StatusIntermediate) - это стандартный набор промежуточных статусов для определенного статуса карточки услуг и определенного типа производства. Вес задавать так - 1-999 (Пока так). Нужно для того, чтобы потом сортировать в соответствии с основными статусами? Складывать вес основного статуса и промежуточного для сортировки? Затем задаются спецификации для каждого промежуточного статуса относящегося к определеннгому статусу карточки услуг. В этой спецификации задается временной норматив и ответственный (роль). Это позволяет контролировать сроки выполнения заказа и видеть на каком этапе происходит отклонение и кто за это отвечает.
